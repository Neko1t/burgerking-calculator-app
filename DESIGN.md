# 汉堡王性价比计算器 - 开源项目设计方案

**版本**: v1.0
**最后更新**: 2026-04-01
**状态**: 已确认

## 项目概述

**目标**: 开源工具，帮助用户在主流团购平台上找到最划算的汉堡王套餐搭配

**核心功能**:
1. 构建以食物为中心的套餐索引（每个食物维护其所在的套餐集合）
2. 用户自由选择菜品组合
3. 计算各食物套餐集合的交集（精确匹配）
4. 推荐高性价比的非精确匹配套餐（多送但更划算的套餐）
5. 保留套餐原始链接/地址

**开源原则**: 零服务器成本、社区驱动、用户可定制

**技术栈**: uni-app + 静态数据CDN + 多种OCR支持

## 核心算法：套餐交集计算

**输入**: 用户选择的多个食物ID列表
**输出**: 按性价比排序的推荐列表

**计算流程**:
1. 获取每个食物的套餐集合
2. 计算交集（精确匹配）
3. 计算子集匹配（高性价比推荐）
4. 计算性价比得分
5. 排序并返回结果

**性价比计算**:
- 精确匹配: `得分 = 单品总价 / 套餐价格`
- 部分匹配: `得分 = (匹配单品总价 / 套餐价格) × 匹配度权重`
